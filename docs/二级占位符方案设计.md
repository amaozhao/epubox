# EPUB 翻译系统 - 二级占位符方案设计

## 1. 背景与问题

### 1.1 当前问题

在 EPUB 翻译过程中，存在以下问题：

1. **标签间空白字符导致过多占位符**
   - HTML 文档中，父子标签之间存在大量 `\r\n`、`\r\r\n`、空格、Tab 等空白字符
   - 当前 `TagPreserver._is_non_translatable()` 方法只检查标签类型，没有检查纯空白字符
   - 导致本应合并为一个占位符的多个相邻标签被分割成多个占位符

2. **pre/code 标签需要完全保护**
   - `<pre>` 标签内的代码格式（缩进、换行）必须保留
   - `<code>` 标签内的代码内容不需要翻译
   - 这些标签内部可能包含 LLM 容易误处理的格式

### 1.2 当前处理流程（有问题）

```
HTML 输入
    ↓
re.split(<[^>]+>) 分割
    ↓
逐 segment 处理
    ↓
_is_non_translatable() 判断
    ↓
问题：空白字符未被识别为非翻译内容
    ↓
生成过多占位符
```

### 1.3 期望结果

```html
<!-- 输入 -->
<div>
  <p>Hello</p>
</div>

<!-- 期望输出（2个占位符） -->
[id0]Hello[id1]

<!-- 实际输出（4个占位符，因为 \n 被错误处理） -->
[id0][id1]Hello[id2][id3]
```

## 2. 方案概述

采用**二级占位符**方案：

| 级别 | 占位符类型 | 示例 | 用途 |
|------|----------|------|------|
| 一级 | 标签占位符 | `[id0]`, `[id1]` | 保护 HTML 标签结构 |
| 二级 | pre/code 占位符 | `[PRE:0]`, `[CODE:0]` | 保护代码块整体 |

### 2.1 核心思想

**将预处理和标签占位符处理分离为两个独立阶段：**

1. 第一阶段：使用 BeautifulSoup 解析器，递归找到所有 `<pre>` 和 `<code>` 标签，整体替换为二级占位符
2. 第二阶段：使用现有的 `TagPreserver` 处理一级标签占位符

### 2.2 如何解决空白字符问题

**问题根源**：当前 `TagPreserver._is_non_translatable()` 没有检查纯空白字符（如 `\n`、`\r\n`、空格等），导致空白字符被错误地当作"可翻译内容"flush出去，破坏了标签合并。

**解决方案**：二级占位符方案

**为什么能解决问题**：

1. **pre/code 内容先被提取**：pre/code 标签内的空白（缩进、换行等）随着标签一起被保护，不再参与后续处理

2. **剩余 HTML 结构更简单**：提取 pre/code 后，剩余的大多是简单的块级标签结构

3. **标签间的空白无害**：即使标签间有 `\n`，被 flush 后只会生成单独的占位符，不会破坏标签合并（因为 flush 后当前组就清空了）

**示例**：
```html
<!-- 原始输入 -->
<div>
  <pre>
    code
  </pre>
  <p>Hello</p>
</div>

<!-- Step 1 提取 pre 后 -->
<div>
  [PRE:0]
  <p>Hello</p>
</div>

<!-- Step 2-5 preserve_tags 后 -->
[id0][PRE:0][id1]Hello[id2]
```

注意：即使 `<div>` 和 `[PRE:0]` 之间的 `\n` 被单独处理（生成 `[id0]`），也不会影响 pre 和 code 的保护。

**已修复空白字符问题**：修改 `TagPreserver.preserve_tags()` 将空白字符（\r\n、空格、Tab）与相邻标签合并为一个占位符，而不是作为独立片段处理。

**原方案（不合并空白）的缺点**：
- 标签间的 `\r\n` 会产生单独的占位符
- 导致 chunk 中的占位符数量增加
- 翻译验证时可能产生索引不匹配的问题

**修复后的方案（合并空白）**：
- `WHITESPACE_PATTERN = re.compile(r'^[\s\r\n\t]+$')`
- 空白字符与标签一起加入 `current_tag_group`
- 标签组合并为一个占位符

## 3. 处理流程（9 步）

### 3.1 提取顺序：先 pre，后 code

由于是递归处理，嵌套情况下：
- `<pre><code>nested</code></pre>` 先匹配外层 `<pre>`，整体替换为 `[PRE:0]`
- 内层 `<code>` 作为字符串内容存在于 pre 中，不会被单独处理

### 3.2 恢复顺序：先 code，后 pre

```
Step 1a: 提取 pre 标签
  extract_pre(html)
  输入: <pre>code</pre><code>x=1</code><p>Hello</p>
  输出: [PRE:0]<code>x=1</code><p>Hello</p>
       + preserved_pre = ["<pre>code</pre>"]

Step 1b: 提取 code 标签
  extract_code(html)
  输入: [PRE:0]<code>x=1</code><p>Hello</p>
  输出: [PRE:0][CODE:0]<p>Hello</p>
       + preserved_code = ["<code>x=1</code>"]

Step 2-5: 正常标签占位符处理（现有逻辑）
  TagPreserver.preserve_tags()
  输入: [PRE:0][CODE:0]<p>Hello</p>
  输出: [PRE:0][CODE:0][id0]Hello[id1]
       + placeholder_mgr.tag_map = {"[id0]": "<p>", ...}

Step 6: 翻译（LLM）
  [PRE:0][CODE:0][id0]Hello[id1]
       →  [PRE:0][CODE:0][id0]你好[id1]

  ⚠️ 翻译提示词必须强调：
  - [PRE:n] 和 [CODE:n] 是占位符，必须原样保留
  - [id:n] 是占位符，必须原样保留

Step 6b: 验证二级占位符完整性（翻译后）
  validate_placeholders(translated, pre_count, code_count)
  输入: [PRE:0][CODE:0][id0]你好[id1]
  如果验证失败：attempt_recovery() 尝试修复格式错误

Step 7: 恢复一级占位符（现有逻辑）
  TagRestorer.restore_tags()
  输入: [PRE:0][CODE:0][id0]你好[id1]
  输出: [PRE:0][CODE:0]<p>你好</p>

Step 8a: 恢复二级占位符 - code（先恢复）
  restore_code(html, preserved_code)
  输入: [PRE:0][CODE:0]<p>你好</p>
  输出: [PRE:0]<code>x=1</code><p>你好</p>

Step 8b: 恢复二级占位符 - pre（后恢复）
  restore_pre(html, preserved_pre)
  输入: [PRE:0]<code>x=1</code><p>你好</p>
  输出: <pre>code</pre><code>x=1</code><p>你好</p>

Step 9: 生成完整翻译文档
```

## 4. 技术实现细节

### 4.1 二级占位符格式设计

| 属性 | 值 |
|------|-----|
| pre 占位符格式 | `[PRE:n]` |
| code 占位符格式 | `[CODE:n]` |
| 分隔符 | `:` |
| 序号起始值 | 0 |
| 序号递增值 | 1 |

**格式特点：**
- 与一级占位符 `[id0]` 格式完全不同，不易混淆
- 方括号格式符合自然语言习惯，LLM 能识别为占位符
- 使用冒号分隔，便于正则匹配

### 4.2 提取 pre/code 标签

```python
def extract_pre_code_blocks(html: str) -> Tuple[str, List[str]]:
    """
    递归提取所有 pre 和 code 标签，替换为占位符

    Args:
        html: 原始 HTML 字符串

    Returns:
        Tuple[处理后HTML, 原始内容列表]
    """
    soup = BeautifulSoup(html, 'html.parser')
    preserved_pre = []
    preserved_code = []

    def process_node(node):
        """
        递归处理节点，将 pre/code 整体替换

        重要实现细节：
        - `list(node.children)` 创建子节点的快照列表
        - 这确保在 `replace_with()` 修改树结构时，迭代不会受影响
        - 处理顺序：从外层到内层（深度优先），确保嵌套标签正确处理
        """
        for child in list(node.children):
            if hasattr(child, 'name') and child.name == 'pre':
                # 先处理子节点（递归）- 处理内层嵌套
                if hasattr(child, 'children'):
                    process_node(child)
                # 然后替换当前 pre 标签
                placeholder = f"[PRE:{len(preserved_pre)}]"
                preserved_pre.append(str(child))
                child.replace_with(BeautifulSoup(placeholder, 'html.parser'))
            elif hasattr(child, 'name') and child.name == 'code':
                # 先处理子节点（递归）- 处理内层嵌套
                if hasattr(child, 'children'):
                    process_node(child)
                # 然后替换当前 code 标签
                placeholder = f"[CODE:{len(preserved_code)}]"
                preserved_code.append(str(child))
                child.replace_with(BeautifulSoup(placeholder, 'html.parser'))
            elif hasattr(child, 'children'):
                process_node(child)

    if soup.body:
        process_node(soup.body)

    return str(soup), preserved_pre, preserved_code
```

### 4.3 恢复 pre/code 标签

**关键：必须按索引顺序从大到小替换，避免被破坏的占位符先被错误匹配**

```python
def restore_pre_code(html: str, preserved: List[str], tag_type: str) -> str:
    """
    恢复 pre 或 code 标签

    重要：按索引从大到小替换，避免以下问题：
    - LLM 可能打乱占位符顺序，如 [PRE:1]...[PRE:0]
    - 如果先替换 [PRE:0]，会错误地替换 [PRE:1] 中的 "0" 部分
    - 从大到小替换可以避免这个问题

    Args:
        html: 包含占位符的 HTML
        preserved: 原始内容列表
        tag_type: "PRE" 或 "CODE"

    Returns:
        恢复后的 HTML
    """
    # 按索引从大到小替换
    for i in range(len(preserved) - 1, -1, -1):
        placeholder = f"[{tag_type}:{i}]"
        html = html.replace(placeholder, preserved[i])
    return html
```

**示例说明：**

```html
# LLM 打乱顺序的输入
"[PRE:1]<p>Hello</p>[PRE:0]"

preserved = ["<pre>A</pre>", "<pre>B</pre>"]  # [PRE:0]=A, [PRE:1]=B

# 正确做法（从大到小替换）
for i in range(len(preserved) - 1, -1, -1):
    html = html.replace(f"[PRE:{i}]", preserved[i])
# Step 1: replace("[PRE:1]", "<pre>B</pre>") → "<pre>B</pre><p>Hello</p>[PRE:0]"
# Step 2: replace("[PRE:0]", "<pre>A</pre>") → "<pre>B</pre><p>Hello</p><pre>A</pre>"

# 最终结果正确：A 在 [PRE:0] 位置，B 在 [PRE:1] 位置
```

### 4.4 完整恢复流程

```python
def restore_all(html: str, placeholder_mgr: PlaceholderManager, extractor: PreCodeExtractor) -> str:
    """
    完整恢复流程：先恢复一级占位符，再恢复二级占位符

    Args:
        html: 翻译后的 HTML
        placeholder_mgr: 一级占位符管理器
        extractor: PreCodeExtractor 实例（包含 preserved_pre 和 preserved_code）

    Returns:
        完整恢复后的 HTML
    """
    # Step 1: 创建恢复器
    restorer = TagRestorer()

    # Step 2: 先恢复一级占位符（标签）
    html = restorer.restore_tags(html, placeholder_mgr)

    # Step 3: 恢复二级占位符（使用 PreCodeExtractor.restore）
    # 内部已实现：先 code，后 pre；从大到小替换
    html = extractor.restore(html)

    return html
```

## 5. 边界情况处理

### 5.1 嵌套 pre/code 标签

```html
<pre><code>nested</code></pre>
```

**处理方式：**
- 递归从外层到内层处理
- 外层 `<pre>` 先被替换为 `[PRE:0]`
- 内层 `<code>` 作为字符串存在于外层内容中，不会被单独处理
- 结果：`[PRE:0]`

**为什么不会丢失：**
- 替换是整体替换，不是逐字替换
- `<pre><code>nested</code></pre>` 整体被保存，然后整体替换为 `[PRE:0]`
- 恢复时完整还原

### 5.2 pre/code 标签内的 id 占位符

```html
<pre>[id0]literal[id1]</pre>
```

**处理流程：**
1. Step 1 提取 pre：`[PRE:0]`（原始内容 `'<pre>[id0]literal[id1]</pre>'`）
2. Step 2-5 处理标签：不再处理（pre 已被挖走）
3. 翻译：不变
4. 恢复：`[PRE:0]` → `<pre>[id0]literal[id1]</pre>`

**重要：id 占位符作为字面量保存在 pre 内容中！**

### 5.3 空的 pre/code 标签

```html
<pre></pre>
<code/>
```

**处理方式：**
- 正常提取和替换
- 恢复时还原为原始空标签

### 5.4 pre/code 标签属性

```html
<pre class="language-python" id="test">
<code data-lang="js">
```

**处理方式：**
- `BeautifulSoup` 解析时保留所有属性
- 整体替换，属性不丢失
- 恢复时完全一致

### 5.5 自闭合 code 标签

```html
<code/>
```

**处理方式：**
- `BeautifulSoup` 将自闭合标签解析为 `NavigableString(' ')` 或类似结构
- `child.name` 可能为 `None`，不会匹配 `'code'`
- 自闭合标签 `<code/>` 不会进入标签组，不会被提取
- 如果需要处理，应在正则层面预处理

**建议**：在 `extract()` 之前，对自闭合标签进行预处理：

```python
def extract(self, html: str) -> str:
    # 预处理：自闭合 code 转为普通 code
    html = re.sub(r'<code\s*/>', '<code></code>', html)
    # ... 后续逻辑
```

### 5.6 大小写不敏感

```html
<PRE>...</PRE>
<Code>...</Code>
```

**处理方式：**
- `BeautifulSoup` 将标签名规范化为小写
- `child.name` 返回 `'pre'` 和 `'code'`（始终小写）
- 代码能正确处理各种大小写组合

### 5.7 深层嵌套

```html
<pre><div><code>nested</code></div></pre>
```

**处理方式：**
- 递归从根节点向叶节点处理
- 最外层 `<pre>` 先被替换为 `[PRE:0]`
- 内层所有内容作为字符串保存在 `<pre>` 内容中
- 结果：`[PRE:0]`

## 6. LLM 输出验证

### 6.1 可能的 LLM 错误

翻译过程中，LLM 可能产生以下错误：

| 错误类型 | 示例 | 原因 |
|---------|------|------|
| 格式变形 | `[PRE:0]` → `[PRE;0]` | 冒号打成分号 |
| 空格添加 | `[PRE:0]` → `[PRE: 0]` | 序号和括号间多了空格 |
| 括号丢失 | `[PRE:0]` → `[PRE:0` | 丢失右括号 |
| 内容修改 | `[PRE:0]` → `PRE:0` | 丢失方括号 |

### 6.2 验证机制

```python
def validate_placeholders(html: str, expected_pre: int, expected_code: int) -> bool:
    """
    验证占位符是否完整

    Returns:
        True 如果所有占位符都存在且格式正确
    """
    pre_found = len(re.findall(r'\[PRE:\d+\]', html))
    code_found = len(re.findall(r'\[CODE:\d+\]', html))

    if pre_found != expected_pre:
        logger.error(f"PRE占位符数量不匹配: 期望{expected_pre}, 实际{pre_found}")
        return False

    if code_found != expected_code:
        logger.error(f"CODE占位符数量不匹配: 期望{expected_code}, 实际{code_found}")
        return False

    return True
```

### 6.3 容错恢复

如果验证失败，尝试容错匹配：

```python
def attempt_recovery(html: str, preserved_pre: List[str], preserved_code: List[str]) -> str:
    """
    尝试恢复可能被破坏的占位符（仅处理格式变形，不处理缺失）

    可修复的模式：
    - [PRE;\d+] → [PRE:\d+]  （分号变冒号）
    - [PRE: \d+] → [PRE:\d+] （多余空格）
    - [CODE;\d+] → [CODE:\d+]
    - [CODE: \d+] → [CODE:\d+]

    不可修复的模式（只能报告错误）：
    - PRE:0 （丢失左方括号）
    - [PRE: （丢失右方括号）
    - [PRE0] （丢失冒号）

    注意：修复后需要重新验证！
    """
    # 修复分号
    html = re.sub(r'\[PRE;(\d+)\]', r'[PRE:\1]', html)
    html = re.sub(r'\[CODE;(\d+)\]', r'[CODE:\1]', html)

    # 修复多余空格
    html = re.sub(r'\[PRE:\s+(\d+)\]', r'[PRE:\1]', html)
    html = re.sub(r'\[CODE:\s+(\d+)\]', r'[CODE:\1]', html)

    return html
```

## 7. 数据结构设计

### 7.1 PreCodeExtractor 类

```python
class PreCodeExtractor:
    """提取和恢复 pre/code 标签"""

    def __init__(self):
        self.preserved_pre: List[str] = []   # 原始 pre 标签列表
        self.preserved_code: List[str] = []  # 原始 code 标签列表

    def extract(self, html: str) -> str:
        """
        提取 pre/code 标签，替换为占位符

        提取顺序：先 pre，后 code（递归处理）

        Returns:
            处理后的 HTML
        """
        soup = BeautifulSoup(html, 'html.parser')
        self.preserved_pre = []
        self.preserved_code = []

        def process_node(node):
            """
            递归处理节点，将 pre/code 整体替换

            重要实现细节：
            - `list(node.children)` 创建子节点的快照列表
            - 这确保在 `replace_with()` 修改树结构时，迭代不会受影响
            - 处理顺序：从外层到内层（深度优先），确保嵌套标签正确处理
            """
            for child in list(node.children):
                if hasattr(child, 'name'):
                    if child.name == 'pre':
                        # 递归处理子节点（处理内层嵌套）
                        if hasattr(child, 'children'):
                            process_node(child)
                        # 然后替换当前 pre 标签
                        placeholder = f"[PRE:{len(self.preserved_pre)}]"
                        self.preserved_pre.append(str(child))
                        child.replace_with(BeautifulSoup(placeholder, 'html.parser'))
                    elif child.name == 'code':
                        # 递归处理子节点（处理内层嵌套）
                        if hasattr(child, 'children'):
                            process_node(child)
                        # 然后替换当前 code 标签
                        placeholder = f"[CODE:{len(self.preserved_code)}]"
                        self.preserved_code.append(str(child))
                        child.replace_with(BeautifulSoup(placeholder, 'html.parser'))
                    elif hasattr(child, 'children'):
                        process_node(child)

        if soup.body:
            process_node(soup.body)

        return str(soup)

    def restore(self, html: str) -> str:
        """
        恢复 pre/code 标签

        恢复顺序：先 code，后 pre
        替换顺序：按索引从大到小，避免子串匹配问题

        Returns:
            恢复后的 HTML
        """
        # 先恢复 code（从大到小）
        for i in range(len(self.preserved_code) - 1, -1, -1):
            html = html.replace(f"[CODE:{i}]", self.preserved_code[i])

        # 后恢复 pre（从大到小）
        for i in range(len(self.preserved_pre) - 1, -1, -1):
            html = html.replace(f"[PRE:{i}]", self.preserved_pre[i])

        return html

    @property
    def pre_count(self) -> int:
        return len(self.preserved_pre)

    @property
    def code_count(self) -> int:
        return len(self.preserved_code)
```

### 7.2 Orchestrator 集成

```python
class TranslatorOrchestrator:
    """翻译编排器"""

    def __init__(self):
        self.pre_code_extractor = PreCodeExtractor()

    def translate_html(self, html: str) -> str:
        # Step 1: 提取 pre/code
        extractor = PreCodeExtractor()
        html = extractor.extract(html)
        pre_count = extractor.pre_count
        code_count = extractor.code_count

        # Step 2-5: 标签占位符处理
        preserver = TagPreserver()
        html, placeholder_mgr = preserver.preserve_tags(html)

        # Step 6: 翻译
        translated = self.llm_translate(html)

        # Step 6b: 验证二级占位符完整性（在恢复之前！）
        # 验证必须在恢复之前进行，因为恢复后 [PRE:n] 和 [CODE:n] 会消失
        if not validate_placeholders(translated, pre_count, code_count):
            translated = attempt_recovery(translated, extractor.preserved_pre, extractor.preserved_code)
            # 重要：attempt_recovery 后必须重新验证！
            if not validate_placeholders(translated, pre_count, code_count):
                logger.error("占位符恢复失败，仍有缺失")

        # Step 7: 恢复一级占位符（id 占位符）
        restorer = TagRestorer()
        translated = restorer.restore_tags(translated, placeholder_mgr)

        # Step 8: 恢复二级占位符（PRE/CODE）
        translated = extractor.restore(translated)

        # Step 9: 生成完整翻译文档
        return translated
```
