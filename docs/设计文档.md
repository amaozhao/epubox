# EPUBox：智能 EPUB 翻译 CLI 工具架构设计

## 1. 项目概述

EPUBox 是一个基于命令行界面的 EPUB 电子书翻译工具，专注于利用大型语言模型（LLM）实现高效、可靠的翻译功能。核心特性包括 **token 消耗优化**、**断点续传支持**，以及 **模块化设计**。

**关键特性**：

- **翻译引擎**：通过 Agno 框架集成 LLM，支持语义感知翻译。
- **技术栈**：Typer (CLI)、ebooklib (EPUB 处理)、Agno (代理系统)、asyncio (异步处理)、tiktoken (token 估算)。
- **占位符保护**：TagPreserver 将 HTML 标签替换为 `[idN]` 占位符，翻译后通过 TagRestorer 恢复。
- **断点续传**：JSON 文件持久化，每个 chunk 翻译后立即保存。
- **二级占位符**：PreCodeExtractor 提取 `<pre>`/`<code>` 标签为 `[PRE:N]`/`[CODE:N]`，避免代码被翻译。

## 2. 系统架构

### 2.1 整体架构

```
EPUB 文件
    ↓
Parser (解析 + 分块)
    ↓
PlaceholderManager (全局占位符映射)
    ↓
HtmlChunker (按 token/占位符数量分块)
    ↓
TranslatorWorkflow (翻译 → 校对 → 修正)
    ↓
Replacer (合并 chunks + 恢复标签)
    ↓
Builder (构建新 EPUB)
```

### 2.2 核心组件

| 组件 | 路径 | 职责 |
|------|------|------|
| **Parser** | `engine/epub/parser.py` | EPUB 解析、TagPreserver 标签替换、HtmlChunker 分块 |
| **PlaceholderManager** | `engine/item/placeholder.py` | 管理 `[idN]` 占位符与原始标签的映射 |
| **TagPreserver** | `engine/item/tag/preserve.py` | 将 HTML 标签替换为 `[idN]` 占位符 |
| **HtmlChunker** | `engine/item/chunker.py` | 按 token 限制和占位符数量分块 |
| **Merger** | `engine/item/merger.py` | 合并 chunks，替换 lang 属性 |
| **Translator** | `engine/agents/translator.py` | LLM 翻译代理 |
| **Proofer** | `engine/agents/proofer.py` | 校对代理，修正错词 |
| **Workflow** | `engine/agents/workflow.py` | 翻译→校对→修正工作流 |
| **TagRestorer** | `engine/item/tag/restore.py` | 将 `[idN]` 恢复为原始标签 |
| **PreCodeExtractor** | `engine/item/precode.py` | 提取/恢复 `<pre>`/`<code>` 为 `[PRE:N]`/`[CODE:N]` |
| **Replacer** | `engine/epub/replacer.py` | 合并 chunks、恢复标签、写文件 |
| **Builder** | `engine/epub/builder.py` | 构建翻译后的 EPUB 文件 |
| **Orchestrator** | `engine/orchestrator.py` | 核心流程协调器 |

## 3. 核心工作流程

### 3.1 解析阶段

1. **解压 EPUB** 到临时目录
2. **PreCodeExtractor** 提取 `<pre>`/`<code>` 为 `[PRE:0]`/`[CODE:0]`
3. **TagPreserver** 将 HTML 标签替换为 `[id0]`/`[id1]`...，返回 `processed_content` 和 `PlaceholderManager`
4. **HtmlChunker** 分块，每个 chunk 最多 15 个占位符、最多 2000 tokens
5. 创建 `EpubItem`，`content` 存原文，`content_with_placeholders` 存占位符版本（JSON 持久化）
6. `Parser.save_json()` 保存解析结果

### 3.2 翻译阶段

1. `Orchestrator.translate_epub()` 加载 JSON
2. 对每个 `EpubItem` 的每个 `Chunk`：
   - `TranslatorWorkflow` 执行翻译→校对→修正
   - **翻译**：LLM 翻译，保留 `[idN]` 占位符
   - **校对**：Proofer 返回 `{"corrections": {"原文片段": "修正后片段"}}`
   - **修正**：按位置从后往前应用修正（避免同一短语多次出现时的位置偏移问题）
   - **后处理**：统一词汇（"您"→"你"、"大型语言模型"→"大语言模型"）和标点
3. **每个 chunk 翻译后立即 `save_json()`**，支持断点续传

### 3.3 恢复阶段

1. **Merger.merge()** 合并所有 chunks 的 `translated` 内容
2. **TagRestorer** 将 `[idN]` 替换回原始标签
3. **PreCodeExtractor.restore()** 将 `[PRE:N]`/`[CODE:N]` 恢复为原始代码
4. 验证 nav 文件结构完整性（支持 NCX 和 XHTML 两种格式）
5. 写入 `item.translated` 并保存到文件

### 3.4 构建阶段

1. `Builder.build()` 从临时目录读取翻译后的文件
2. 打包为新的 EPUB 文件（`{name}-cn.epub`）

## 4. 占位符系统

### 4.1 三层架构

```
原始 HTML: <p>Hello <b>World</b></p>
    ↓ TagPreserver
占位符版本: [id0]Hello [id1]World[id2]
    ↓ 翻译
翻译占位符: [id0]你好 [id1]世界[id2]
    ↓ TagRestorer
翻译结果: <p>你好 <b>世界</b></p>
```

### 4.2 占位符格式

| 类型 | 格式 | 说明 |
|------|------|------|
| 标签占位符 | `[id0]`, `[id1]`, ... | 全局唯一，文档级别连续编号 |
| Pre 占位符 | `[PRE:0]`, `[PRE:1]`, ... | `<pre>` 标签二级保护 |
| Code 占位符 | `[CODE:0]`, `[CODE:1]`, ... | `<code>` 标签二级保护 |

### 4.3 TagPreserver 合并策略

相邻的标签和空白**合并为一个占位符**，例如：
- `<p>Hello</p>` → `[id0]Hello[id1]`
- `<p><b>Bold</b></p>` → `[id0]<b>Bold</b>[id1]`（内嵌标签不单独占位）

## 5. 数据模型

### 5.1 EpubItem

```python
class EpubItem(BaseModel):
    id: str                          # 文件路径标识
    path: str                        # 文件系统路径
    content: str                     # 原始 HTML 内容
    content_with_placeholders: str    # 占位符版本（解析时生成）
    translated: Optional[str]         # 翻译恢复后的内容
    placeholder: Dict[str, str]      # 占位符→原始标签映射
    chunks: List[Chunk]              # 分块列表
    preserved_pre: List[str]         # 提取的 <pre> 内容
    preserved_code: List[str]         # 提取的 <code> 内容
```

### 5.2 Chunk

```python
class Chunk(BaseModel):
    name: str                        # UUID 前 8 位，唯一标识
    original: str                    # 原始文本（含占位符）
    translated: Optional[str]        # 翻译后文本（含占位符）
    status: TranslationStatus        # PENDING/TRANSLATED/UNTRANSLATED/COMPLETED
    tokens: int                      # token 估算
    global_indices: List[int]        # 全局占位符索引
    local_tag_map: Dict[str, str]   # 局部占位符→标签映射
```

## 7. 翻译工作流详解

```
┌──────────────────────────────────────────────────────┐
│                  TranslatorWorkflow                   │
│                                                       │
│  Step 1: translate                                   │
│    Input: Chunk (original + local_tag_map)           │
│    LLM: 保留 [idN] 占位符，翻译文本内容               │
│    Output: Chunk (translated + status)              │
│                     ↓                                 │
│  Step 2: proofread                                    │
│    Input: translated text                            │
│    LLM: 找出错词，返回 {"corrections": {...}}         │
│    Output: ProofreadingResult                        │
│                     ↓                                 │
│  Step 3: apply_corrections                           │
│    按位置从后往前替换修正短语                         │
│    后处理：统一词汇、标点                             │
│    Output: Chunk (translated + COMPLETED)            │
└──────────────────────────────────────────────────────┘
```

## 8. 断点续传机制

1. **JSON 持久化**：解析后保存 `{name}.json`，包含所有 items、chunks、placeholder 映射
2. **即时保存**：每个 chunk 翻译后立即 `save_json()`
3. **重新加载**：重启时 `load_json()` 恢复状态，跳过已翻译的 chunks
4. **手动翻译报告**：翻译失败的 chunk 记录到 `manual_translation_report.json`

---

## 9. 性能与可靠性

- **Token 限制**：每个 chunk ≤ 2000 tokens，最多 15 个占位符
- **并发控制**：asyncio 异步并发翻译
- **重试机制**：翻译和校对步骤均支持最多 3 次重试
- **回退模型**：主模型失败时自动切换备用模型
- **优雅降级**：翻译失败保留原文保结构
