# EPUB 翻译系统重构文档

## 目标

实现"先分块后替换"的翻译流程：
- 按 HTML 标签边界分块，保证标签对完整
- 每个 chunk 独立做标签占位符替换
- 翻译后立即恢复标签占位符
- 保留 pre/code/style 全局占位符到最终恢复

---

## 重构流程

```
1. PreCodeExtractor.extract()
   → [PRE:n], [CODE:n], [STYLE:n]  (全局占位符，跨所有 chunk)

2. HtmlChunker.chunk_by_html_tags()
   → 原始 HTML 片段列表（每个片段仍含 [PRE:n] 等）

3. 对每个 chunk 独立执行：
   ├── TagPreserver.preserve_tags()
   │    → [id0], [id1]... (局部占位符，每个 chunk 独立)
   ├── 翻译
   └── TagRestorer.restore_tags(translated, chunk.local_tag_map)
        → [idN] → 原始标签
        恢复后文本不包含 [idN]，但仍含 [PRE:n]

4. 所有 chunk 翻译完成：
   ├── 合并成完整翻译文本
   ├── PreCodeExtractor.restore()  → [PRE:n] → 原始 pre/code/style
   └── HTML 完整性检查
```

## 核心设计决策

### 1. 分块策略

| 分类 | 标签 | 说明 |
|------|------|------|
| 安全分割点 | `</p>`, `</div>`, `</h1-6>`, `</blockquote>` 等 | 块级边界，可安全分割 |
| 容器级分割点 | `</table>`, `</ul>`, `</ol>` | 万不得已时，整体作为 chunk |
| 禁止分割点 | `</tr>`, `</td>`, `</th>`, `</li>` | 会破坏表格/列表结构 |
| 内联标签 | `</a>`, `</span>`, `</em>` 等 | 禁止在标签对内部分割 |

**句子边界 fallback**：超长 chunk 在句子边界（`.!。！？`后）分割。

### 2. 占位符设计

**三层占位符**：

| 占位符 | 作用域 | 保留到 |
|--------|--------|--------|
| `[id0]`, `[id1]`... | 单个 chunk | 该 chunk 翻译后立即恢复 |
| `[PRE:n]` | 整个文档 | 最终恢复 |
| `[CODE:n]` | 整个文档 | 最终恢复 |
| `[STYLE:n]` | 整个文档 | 最终恢复 |

### 3. 翻译工作流

```
Phase 1: 正常翻译 + 占位符验证
   ↓ 失败
Phase 2: Token Alignment Fallback
   (移除占位符 → 翻译纯净文本 → 词级对齐重新插入)
   ↓ 失败
Phase 3: 返回原文 (UNTRANSLATED)
```

## 阻塞问题

### 问题 1：`chunk_by_html_tags()` 从未被调用

**位置**：`parser.py`

**现状**：`parser.py` 使用旧流程（先全局 preserve_tags，后 chunk()）

**修复**：改为先 `chunk_by_html_tags()` 分块，后对每个 chunk 单独 `preserve_tags()`

### 问题 2：Phase 2 从未调用

**位置**：`workflow.py`

**现状**：`aligner.py` 有实现但从未被 `workflow.py` 调用

**修复**：在 Phase 1 失败后调用 `token_alignment_fallback()`

### 问题 3：`local_tag_map` 未使用

**位置**：`chunk.py`, `orchestrator.py`

**现状**：`local_tag_map` 被填充但翻译时传入全局 `placeholder_mgr`

**修复**：翻译时传入 `chunk.local_tag_map`，恢复时用它

### 问题 4：容器标签未整体分割

**位置**：`chunker.py`

**现状**：`_is_inside_container()` 未实现，`</table>` 可能在 table 内部分割

**修复**：实现 `_is_inside_container()` 检查

### 问题 5：Phase 2 未处理 `[PRE:n]`

**现状**：`aligner._remove_placeholders()` 只移除 `[idN]`，不处理 `[PRE:n]`

**修复**：对齐前先移除 `[PRE:n]`，对齐后恢复

### 修复 4：Phase 2 处理 `[PRE:n]`

```python
async def token_alignment_fallback(original, local_tag_map, preserved_pre):
    # 1. 移除 [PRE:n]/[CODE:n]/[STYLE:n]，记录位置
    temp, pre_map = remove_pre_placeholders(original, preserved_pre)
    # 2. 移除 [idN]
    clean = remove_id_placeholders(temp, local_tag_map.keys())
    # 3. 翻译
    translated = await translate(clean)
    # 4. 对齐重新插入 [idN]
    aligned = align_placeholders(temp, translated, local_tag_map.keys())
    # 5. 恢复 [PRE:n]
    return restore_pre_placeholders(aligned, pre_map)
```

### 修复 5：容器标签检查

```python
def _is_inside_container(self, html: str, pos: int) -> bool:
    """检查位置是否在容器标签内部"""
    before, after = html[:pos], html[pos:]
    for container in CONTAINER_TAGS:
        opens = list(re.finditer(f'<{container}[^>]*>', before))
        if opens and re.search(f'</{container}>', after):
            return True
    return False
```
