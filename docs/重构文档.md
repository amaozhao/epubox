# EPUB 翻译系统重构文档

## 目标

实现**容器标签完整 + 栈验证**的翻译架构：
- XML 声明和 head 内容不翻译，原样保留
- 容器标签（nav, ol, ul, div 等）不跨 Chunk 切开
- 翻译过程中用栈追踪验证标签配对
- 最终合并后验证所有标签闭合

---

## 核心架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                        解析阶段                                       │
│  识别文件类型 (HTML/NCX/XML)                                        │
│  生成 Chunk 列表：                                                   │
│    - Chunk[0]: XML声明 + DOCTYPE + <html> + <head> + <body>        │
│    - Chunk[1...N]: 翻译内容 (容器标签完整)                           │
│    - Chunk[N+1]: </body></html> (如需要)                            │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│                      翻译循环                                        │
│  for chunk in chunks:                                               │
│    if 不需要翻译: continue                                           │
│    translate(chunk) → 栈验证 (tag_mismatch, unexpected_close)        │
│    if 栈验证失败: chunk.status = UNTRANSLATED                        │
│    save JSON（每完成一个即持久化）                                    │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────────┐
│                      合并阶段                                        │
│  按 chunks 顺序拼接所有内容                                          │
│  最终验证：栈必须为空（除原文就有的未闭合标签）                      │
│  如果验证失败：用原文代替翻译失败的 chunk                              │
└─────────────────────────────────────────────────────────────────────┘
```

## 文件类型识别

| 文件类型 | 结构特征 | 第一个 Chunk |
|---------|---------|-------------|
| XHTML | `<html>`, `<head>`, `<body>` | XML + DOCTYPE + html + head + body |
| NCX | `<ncx>`, `<navMap>` | XML + ncx + navMap |
| 纯 XML | 其他 | XML + 根元素 |

## Chunk 生成策略

### 3 类 Chunk

| 类型 | 翻译 | 内容 |
|-----|-----|------|
| 前缀 Chunk | 不 | XML声明 + DOCTYPE + `<html>` + `<head>` + `<body>`开头 |
| 内容 Chunk | 是 | `<nav>`, `<ol>`, `<p>`, `<li>` 等 |
| 后缀 Chunk | 不 | `</body></html>` |

### 容器标签（不能切开）

```python
CONTAINER_TAGS = {
    'nav', 'ol', 'ul', 'dl', 'div', 'section', 'article',
    'header', 'footer', 'main', 'aside', 'figure',
    'table', 'thead', 'tbody', 'tfoot', 'tr',
}
```

**注意：`html`, `body`, `head` 不在容器列表中，它们只出现在前缀 Chunk。**

### 分块规则

```
1. 递归遍历 HTML 树
2. 对于容器标签（nav, ol, ul, div 等）：
   - 整个容器作为 1 个 chunk（即使超限也不拆分）
3. 对于叶子标签（li, p, h1-h6, span 等）：
   - 贪心累积到接近 token_limit
4. 前缀 Chunk：收集 xml + doctype + html + head + body 开头
5. 后缀 Chunk：如果原文不是以 </body> 结尾，收集 </body></html>
```

### 算法伪代码

```
function chunk_tree(node, token_limit):
    result = []

    for child in node.children:
        if is_container_tag(child.tag):
            # 容器标签：不能拆分，整个作为 1 个 chunk
            result.append(create_chunk([child]))
        else:
            # 叶子节点：累积
            result.extend(chunk_leaves([child], token_limit))

    return result

function chunk_leaves(nodes, token_limit):
    """累积叶子节点到 chunk"""
    result = []
    current = []
    current_tokens = 0

    for node in nodes:
        node_tokens = count_tokens(node.to_html())

        if current_tokens + node_tokens <= token_limit:
            current.append(node)
            current_tokens += node_tokens
        else:
            if current:
                result.append(create_chunk(current))
            current = [node]
            current_tokens = node_tokens

    if current:
        result.append(create_chunk(current))
    return result

function is_container_tag(tag):
    return tag.lower() in CONTAINER_TAGS
```

## 栈验证逻辑

### 容器标签 vs 叶子标签

| 类型 | 标签示例 | 跨 Chunk 未闭合 |
|-----|---------|---------------|
| 容器标签 | `nav`, `ol`, `ul`, `div`, `body` 等 | ✅ 正常 |
| 叶子标签 | `p`, `li`, `h1-h6`, `span`, `em`, `strong` 等 | ❌ 错误 |

**叶子标签必须在同一个 Chunk 内闭合。**

### 验证规则

| 情况 | 是否错误 |
|------|---------|
| 容器标签在栈中但未关闭（跨 Chunk） | 正常 |
| 叶子标签在栈中但未闭合 | 错误 |
| `tag_mismatch` - 关闭标签与打开标签不匹配 | 错误 |
| `unexpected_close` - 关闭标签无对应打开 | 错误 |

### 每翻译一个 Chunk 后验证

```python
validator = HtmlValidator()
valid, errors = validator.validate_chunk(translated, chunk_index, chunk_name)

# 检查：
# 1. tag_mismatch - 标签不匹配
# 2. unexpected_close - 意外闭合
# 3. 叶子标签未闭合（如 <p> 没有 </p>）
```

### 最终合并验证

```python
validator = HtmlValidator()
valid, errors = validator.validate_merged(merged_chunks, chunk_names)

# 合并后栈必须为空
```

## 合并逻辑

```python
def merge_chunks(chunks):
    parts = []
    for chunk in chunks:
        if chunk.status == UNTRANSLATED:
            parts.append(chunk.original)
        elif chunk.translated:
            parts.append(chunk.translated)
        else:
            parts.append(chunk.original)

    merged = "".join(parts)

    # 最终验证
    validator = HtmlValidator()
    valid, errors = validator.validate_merged(merged)
    if not valid:
        merged = "".join(c.original for c in chunks)

    return merged
```

## 边缘情况

### 1. 叶子标签未闭合

```html
Chunk: <p>Hello  (缺少 </p>)
```

**处理：** 叶子标签（p, li, h1-h6 等）在同一个 Chunk 内必须闭合。未闭合标记为翻译失败。

### 2. 容器标签跨 Chunk

```html
Chunk[1]: <body><nav epub:type="toc">...
Chunk[2]: ...</nav></body>
```

**处理：** 容器标签（nav, ol, ul, div, body 等）可以跨 Chunk，正常的。

### 3. 自闭合标签

```html
<div id="page1" epub:type="pagebreak" />
<img src="image.png" />
```

**处理：** 自闭合标签不进入栈，不影响验证。

### 4. 空 Chunk

```html
<li></li>
```

**处理：** 正常，栈为空，不是错误。

### 5. 嵌套容器

```html
<nav><ol><li><ol><li>Nested</li></ol></li></ol></nav>
```

**处理：** 栈正确追踪嵌套结构，关闭时反向出栈。

### 6. NCX 文件

```xml
<?xml version="1.0"?>
<ncx><navMap><navPoint>...</navPoint></navMap></ncx>
```

**处理：** NCX 的 `<ncx>` 和 `<navMap>` 作为容器标签处理。

### 7. 表格语义结构

```html
<table>
  <thead><tr><th>Header</th></tr></thead>
  <tbody><tr><td>Data</td></tr></tbody>
</table>
```

**处理：** `thead`, `tbody`, `tfoot` 作为容器标签，整个 `<table>` 或各自的 `<thead>/<tbody>` 作为 1 个 chunk。

**注意：** BeautifulSoup 会自动插入 `<tbody>`，但我们的 tree.py 使用 `html.parser` 不会。需要确保解析器不会改变原始结构。

### 8. 原文有未闭合标签

```html
<body><p>Hello
（没有 </p></body>）
```

**处理：** 如果原文就有未闭合标签（如省略了可选的闭合标签），对比原文和译文的未闭合数量。如果译文有额外的未闭合，标记为翻译失败。

### 9. 解析器选择

**要求：** 解析器不能自动修复或改变原始 HTML 结构。

| 解析器 | 自动修复 | 适用 |
|--------|---------|------|
| BeautifulSoup | ✅ 会自动修复 | 不适用 |
| lxml XML 解析器 | ❌ 严格 | 适用 XHTML/NCX |
| html.parser | ❌ 宽松修复 | 适用（当前 tree.py 使用） |

**当前 `tree.py` 使用 Python 内置 `html.parser`，不会自动插入 `<tbody>` 等。**

## 验证清单

- [ ] XML 声明正确保留
- [ ] DOCTYPE 正确保留
- [ ] 容器标签完整（nav, ol, ul, div 等）
- [ ] 叶子标签闭合（p, li, h1-h6 等）
- [ ] `tag_mismatch` 正确检测
- [ ] `unexpected_close` 正确检测
- [ ] 跨 Chunk 的容器标签不误报

---

## 验收标准

1. **容器标签不切开** - `<nav>`, `<ol>` 等完整在一个 Chunk 中
2. **叶子标签必须闭合** - `<p>`, `<li>` 等在同一 Chunk 内必须有对应的闭合标签
3. **栈验证正确** - `tag_mismatch` 和 `unexpected_close` 正确检测
4. **跨 Chunk 不误报** - 容器标签（nav, ol, body 等）跨 Chunk 是正常的
5. **翻译失败容错** - 翻译失败的 Chunk 用原文代替
