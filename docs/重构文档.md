# EPUB 翻译系统重构文档

## 目标

实现**树状递归分块 + 语义合并**的翻译架构：
- 按 HTML 树状结构递归分块，保证标签对完整
- 贪心合并符合 token 限制的最大子标签节点
- 通过 xpath 确定性回填翻译结果
- Manifest 持久化支持断点续传

---

## 核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                        解析阶段                             │
│  HTML → 树状解析 → 递归分块 → Chunk列表（带 xpath）        │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      持久化状态                             │
│  加载 Manifest（如存在）→ 恢复进度 → 跳过已完成 Chunk       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      翻译循环                               │
│  for chunk in chunks:                                       │
│    if COMPLETED: continue                                   │
│    translate(chunk) → xpath 回填                            │
│    save Manifest（每完成一个即持久化）                       │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                      重建阶段                               │
│  按 xpath 位置重组 → 输出完整 HTML                          │
└─────────────────────────────────────────────────────────────┘
```

## 核心数据结构

### ChunkState

```python
@dataclass
class ChunkState:
    chunk_id: str                      # hash(xpath + original_html[:100])，用于校验
    xpath: str                         # 在文档中的唯一路径，如 "html/body/div[0]"
    original_html: str                 # 原始 HTML 子树（完整标签树）

    status: ChunkStatus                # PENDING | COMPLETED | FAILED
    translated_html: str | None        # 翻译后 HTML

    token_count: int                  # 估算 token 数
    error: str | None                  # 失败原因

    # 翻译内容
    texts: list[str]                   # 需要翻译的纯文本列表
    text_positions: list[dict]         # 每个文本的位置信息
    context_before: str | None         # 前一个 Chunk 的上下文（不翻译）
    context_after: str | None          # 后一个 Chunk 的上下文（不翻译）

    # 时间戳
    created_at: str
    updated_at: str
    completed_at: str | None
```

### TranslationManifest

```python

class TranslationManifest:
```

```python

class TranslationResponse:
```

## 分块算法

### 核心原则

1. **递归遍历树**：按节点递归，不是按 token 硬切
2. **贪心合并**：尽可能合并子节点直到 token 达限
3. **语义边界**：只在闭合标签处分割，不在标签中间切
4. **单个节点超限则递归**：继续处理它的子节点

### 算法伪代码

```
function chunk_node(node, current_tokens, current_nodes):
    node_tokens = estimate_tokens(node)

    if current_tokens + node_tokens <= SAFE_LIMIT:
        # 可以加入当前 Chunk
        current_nodes.append(node)
        return current_tokens + node_tokens, current_nodes
    else:
        # 保存当前 Chunk
        if current_nodes is not empty:
            yield create_chunk(current_nodes)

        # 如果单个节点就超限，递归处理子节点
        if node_tokens > SAFE_LIMIT:
            if is_leaf_node(node):
                # 极端情况：单个文本节点超限
                # 需要拆分文本节点
                yield from split_text_node(node)
                return 0, []
            else:
                result = []
                for child in node.children:
                    result.extend(chunk_node(child, 0, []))
                return 0, result  # 递归子节点不累积到当前 chunk
        else:
            # 重置，开始新 Chunk
            return chunk_node(node, 0, [node])

function chunk_tree(root):
    chunks = []
    for child in root.children:
        tokens, nodes = chunk_node(child, 0, [])
        if nodes:
            chunks.extend(create_chunk(nodes))
    return chunks
```

### 分块示例

```
<div>                          ← 根节点，超限
  ├── <p>第1段</p>             ← 开始累积
  ├── <p>第2段</p>             ← 累积后 token=600，可接受
  ├── <p>第3段</p>             ← 累积后 token=900，可接受
  ├── <p>第4段</p>             ← 累积后 token=1200，超限！
  │                            ← → 前3个<p>合为一个 Chunk
  ├── <p>第5段</p>             ← 重新开始累积，token=300，可接受
  └── <div>超长内容...</div>   ← 本身超限 → 递归处理
```

**结果**：
```
Chunk 1: <div><p>第1段</p><p>第2段</p><p>第3段</p></div>
Chunk 2: <div><p>第4段</p><p>第5段</p></div>
Chunk 3: <div>超长内容...</div>  （递归处理，内部可能再分）
```

## Token 计算

### 安全边界

```python
MAX_TOKEN_LIMIT = 1000
SAFE_LIMIT = int(MAX_TOKEN_LIMIT * 0.85)  # 预留 15% 余量
```

### 估算函数

```python
def estimate_tokens(text: str) -> int:
    """估算 token 数（使用 tiktoken）"""
    try:
        tokenizer = tiktoken.encoding_for_model("gpt-3.5-turbo")
    except KeyError:
        tokenizer = tiktoken.get_encoding("cl100k_base")
    return len(tokenizer.encode(text))
```

### 超长文本节点处理

```python
MAX_TEXT_NODE_TOKENS = 500

def split_text_node(text_node):
    """超长文本节点需要拆分"""
    words = text_node.text.split()
    chunks = []
    current = []
    current_tokens = 0

    for word in words:
        word_tokens = estimate_tokens(word)
        if current_tokens + word_tokens > MAX_TEXT_NODE_TOKENS:
            chunks.append(" ".join(current))
            current = [word]
            current_tokens = word_tokens
        else:
            current.append(word)
            current_tokens += word_tokens

    if current:
        chunks.append(" ".join(current))
    return chunks
```

## xpath 回填机制

### 位置标识格式

```
xpath:tag:text_index
示例: "p[0]:text:0"  表示 p[0] 节点的第 0 个文本子节点
```

### xpath 格式校验

```python
import re

XPATH_PATTERN = re.compile(r'^([a-zA-Z][a-zA-Z0-9]*)(\[\d+\])*(:(text|attr))?:\d+$')

def is_valid_relative_xpath(rel_xpath: str) -> bool:
    """校验相对 xpath 格式是否合法"""
    return XPATH_PATTERN.match(rel_xpath) is not None


def parse_relative_xpath(rel_xpath: str) -> tuple[str, str, int]:
    """
    解析相对 xpath
    格式: "tag[index]:type:child_index"
    示例: "p[0]:text:0" → ("p[0]", "text", 0)
          "div[1]:attr:2" → ("div[1]", "attr", 2)

    Returns:
        (tag_with_index, type, child_index)

    Raises:
        ValueError: 格式不正确时
    """
    if not is_valid_relative_xpath(rel_xpath):
        raise ValueError(f"Invalid xpath format: {rel_xpath}")

    # 分割：最后一部分是索引，前面是 tag 和 type
    last_colon = rel_xpath.rfind(":")
    if last_colon == -1:
        raise ValueError(f"Invalid xpath format: {rel_xpath}")

    index_part = rel_xpath[last_colon + 1:]
    prefix = rel_xpath[:last_colon]

    # 再次分割，拿到 tag 和 type
    # 格式可能是 "tag[index]:type" 或 "tag[index]"
    type_colon = prefix.rfind(":")
    if type_colon == -1:
        tag_part = prefix
        node_type = "text"  # 默认是文本节点
    else:
        tag_part = prefix[:type_colon]
        node_type = prefix[type_colon + 1:]

    try:
        return tag_part, node_type, int(index_part)
    except ValueError:
        raise ValueError(f"Invalid xpath format: {rel_xpath}")
```

### 回填实现

```python
class TreeNode:
    """树节点"""

    def find_by_xpath(self, xpath: str) -> "TreeNode | None":
        """根据完整 xpath 查找节点"""
        # 例如 "html/body/div[0]"
        parts = xpath.split("/")
        current = self
        for part in parts:
            if not part:
                continue
            # part 可能是 "div" 或 "div[0]"
            match = re.match(r'^([a-zA-Z]+)\[?(\d*)\]?$', part)
            if not match:
                return None
            tag_name, index = match.groups()
            children = [c for c in current.children if c.tag == tag_name]
            if not children:
                return None
            idx = int(index) if index else 0
            if idx >= len(children):
                return None
            current = children[idx]
        return current

    def find_child(self, relative_path: str) -> "TreeNode | None":
        """根据相对路径查找子节点"""
        # 例如 "p[0]" 或 "div[1]"
        match = re.match(r'^([a-zA-Z]+)\[(\d+)\]$', relative_path)
        if not match:
            return None
        tag_name, index = match.groups()
        children = [c for c in self.children if c.tag == tag_name]
        if not children:
            return None
        idx = int(index)
        if idx >= len(children):
            return None
        return children[idx]

    def replace_text_at_index(self, text_index: int, new_text: str):
        """替换第 text_index 个文本节点的内容"""
        text_nodes = self._collect_text_nodes()
        if text_index >= len(text_nodes):
            raise IndexError(f"Text index {text_index} out of range")
        text_nodes[text_index].content = new_text

    def _collect_text_nodes(self) -> list["TreeNode"]:
        """收集所有文本子节点"""
        result = []
        for child in self.children:
            if child.is_text_node():
                result.append(child)
            else:
                result.extend(child._collect_text_nodes())
        return result


def backfill(tree: TreeNode, xpath: str, translations: list[dict]):
    """确定性查表回填，不依赖正则匹配"""
    root = tree.find_by_xpath(xpath)
    if root is None:
        raise ValueError(f"Cannot find node at xpath: {xpath}")

    for item in translations:
        rel_xpath = item["xpath"]
        translated = item["translated"]

        # 解析相对路径
        tag_part, node_type, text_index = parse_relative_xpath(rel_xpath)

        # 找到对应节点
        node = root.find_child(tag_part)
        if node is None:
            raise ValueError(f"Cannot find child at relative xpath: {rel_xpath}")

        # 替换文本
        if node_type == "text":
            node.replace_text_at_index(text_index, translated)
        elif node_type == "attr":
            node.set_attribute(text_index, translated)  # pragma: nocover
```

## 上下文传递

### 目的

解决 Chunk 边界处上下文丢失导致的翻译不连贯问题。

### 实现

```python
# 构建 Chunk 时设置上下文（使用原始文本，非翻译后文本）
for i, chunk in enumerate(chunks):
    if i > 0:
        # 前一个 Chunk 的最后一个原始文本节点
        prev_texts = chunks[i-1].texts
        chunk.context_before = prev_texts[-1] if prev_texts else None

    if i < len(chunks) - 1:
        # 后一个 Chunk 的第一个原始文本节点
        next_texts = chunks[i+1].texts
        chunk.context_after = next_texts[0] if next_texts else None
```

**注意**：`context_before` 和 `context_after` 使用的是**原始文本**，不是翻译后的文本。目的是让 LLM 在翻译时能了解上下文，但不需要翻译上下文本身。

### LLM 提示词

```
You are translating an HTML fragment.

Original text from previous chunk (for reference only, do NOT translate):
"{context_before}"

Original text from next chunk (for reference only, do NOT translate):
"{context_after}"

Rules:
1. Translate ONLY the texts listed in this chunk
2. Keep all HTML tags exactly as-is
3. Do not add, remove, or reorder any tags
4. Maintain consistent style and tone with surrounding content
```

### 原子写入

```python
import tempfile
import atomicwrites

def save_manifest(manifest):
    """原子写入，避免写入中途崩溃导致文件损坏"""
    temp_path = manifest.path + ".tmp"
    with atomicwrites.atomic_write(temp_path, overwrite=True) as f:
        f.write(json.dumps(manifest.to_dict(), indent=2))
    os.replace(temp_path, manifest.path)
```

### 断点续传流程

```
1. 解析 HTML → 生成 chunks

2. 加载 manifest（如果存在）：
   for each chunk in manifest:
       current_hash = hash(chunk.xpath + current_original_html)
       if current_hash != chunk.chunk_id:
           # 原文已变，标记为 PENDING
           chunk.status = PENDING
           chunk.translated_html = None

3. 遍历 chunks：
   if chunk.status == COMPLETED:
       continue  # 跳过已完成的
   else:
       chunk.status = PENDING
       translate(chunk)
       save_manifest()  # 每完成一个立即持久化
```

## 翻译工作流

### 单个 Chunk 翻译流程

```
1. 构建 TranslationRequest
   - xpath: Chunk 的唯一路径
   - html: 原始 HTML（仅用于参考结构）
   - texts: 需要翻译的文本列表
   - positions: 每个文本的位置映射
   - context_before/after: 上下文
   - glossaries: 术语表

2. 调用 LLM 翻译

3. 验证响应格式
   - JSON 是否合法
   - translations 数量是否匹配 texts 数量
   - 每个 xpath 是否有效
   - 翻译文本是否为空

4. 回填到树
   - 按 xpath 找到对应节点
   - 替换文本内容

5. 更新 ChunkState
   - status = COMPLETED
   - translated_html = 翻译后的 HTML
   - completed_at = 当前时间
```

### 验证响应格式

```python
def validate_translation_response(response, expected_texts):
    """验证 LLM 返回的正确性"""
    try:
        data = json.loads(response)
    except json.JSONDecodeError:
        raise TranslationFormatError("Invalid JSON")

    # 1. 检查数量匹配
    if len(data["translations"]) != len(expected_texts):
        raise TranslationIncompleteError(
            f"Expected {len(expected_texts)} translations, "
            f"got {len(data['translations'])}"
        )

    # 2. 检查相对 xpath 有效
    for item in data["translations"]:
        if not is_valid_relative_xpath(item["xpath"]):
            raise TranslationFormatError(f"Invalid xpath: {item['xpath']}")

    # 3. 检查非空
    for item in data["translations"]:
        if not item["translated"].strip():
            raise TranslationEmptyError(f"Empty translation for {item['xpath']}")

    return data
```

## Nav/Toc 文件处理

### 特殊性

- `<navPoint>` 是核心语义单元
- `<text>` 是唯一需要翻译的文本
- `<content src="..."/>` 是链接，不需要翻译但要保留

### Chunk 结构

```python
class NavChunk:
    xpath: str  # 指向 navPoint

    # 可翻译部分
    texts_to_translate = [
        {"xpath": "navLabel/text:0", "text": "Chapter 1"}
    ]

    # 需要保留的非翻译部分
    preserved = {
        "content_src": "Chapter1.xhtml"
    }
```

### 处理流程

Nav/Toc 文件使用相同的递归分块算法，但节点类型不同：

```
<navMap>
  <navPoint id="np1">                    ← navPoint 1，token=100，可累积
    <navLabel><text>Chapter 1</text></navLabel>
    <content src="Chapter1.xhtml"/>
  </navPoint>
  <navPoint id="np2">                    ← navPoint 2，token=100，累积200，可接受
    <navLabel><text>Chapter 2</text></navLabel>
    <content src="Chapter2.xhtml"/>
  </navPoint>
  <navPoint id="np3">                    ← navPoint 3，token=100，累积300，超限！
    <navLabel><text>Chapter 3</text></navLabel>
    <content src="Chapter3.xhtml"/>
  </navPoint>
</navMap>
```

**结果**：
```
Chunk 1: <navMap> 包含 navPoint 1 和 2
Chunk 2: <navPoint id="np3"> 整体
```

**注意**：`content` 元素不需要翻译，但在遍历时需要保留其 `src` 属性。`text` 元素的内容是唯一需要翻译的部分。

## 错误处理与重试策略

### 重试机制

```python
MAX_RETRIES = 3
RETRY_DELAY = 2  # 秒

async def translate_chunk_with_retry(chunk: ChunkState) -> ChunkState:
    """带重试的翻译"""
    for attempt in range(MAX_RETRIES):
        try:
            return await translate_chunk(chunk)
        except TranslationError as e:
            if attempt < MAX_RETRIES - 1:
                logger.warning(f"Translation failed (attempt {attempt + 1}), retrying...")
                await asyncio.sleep(RETRY_DELAY * (attempt + 1))
                continue
            else:
                # 所有重试都失败
                chunk.status = ChunkStatus.FAILED
                chunk.error = str(e)
                return chunk
```

### 错误类型与处理

| 错误类型 | 说明 | 处理方式 |
|---------|------|---------|
| `TranslationFormatError` | LLM 返回格式错误 | 重试 |
| `TranslationIncompleteError` | 翻译数量不完整 | 重试 |
| `TranslationEmptyError` | 翻译结果为空 | 重试 |
| `NetworkError` | 网络错误 | 重试 |
| `RateLimitError` | API 限流 | 等待后重试 |
| `TokenLimitExceeded` | Token 超限 | 拆分为更小的 chunk |

## 验收标准

1. **结构完整性**：每个 Chunk 都是完整的 HTML 子树，标签永不配对丢失
2. **Token 限制**：每个 Chunk 的 token 数不超过 SAFE_LIMIT
3. **断点续传**：翻译中断后，重新启动只重新翻译未完成的 Chunk
4. **上下文连贯**：Chunk 边界处有上下文传递
5. **回填准确**：通过 xpath 查表回填，不依赖正则匹配
6. **Nav/Toc 支持**：导航文件使用相同架构处理
7. **术语表应用**：翻译时正确应用术语表
