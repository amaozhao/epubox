# EPUBox 设计文档

## 系统架构

EPUBox 是一个基于 FastAPI 的 EPUB 电子书翻译服务。系统主要由以下几个核心组件组成：

### 1. EPUB 处理器 (EpubProcessor)

EPUB处理器负责处理EPUB文件的整体流程：

- **文件准备**：
  - 复制原始文件到工作目录
  - 加载EPUB文件到内存

- **内容处理**：
  - 提取HTML内容和NCX导航文件
  - 使用HTML处理器翻译内容
  - 更新处理后的内容到EPUB文件

- **文件保存**：
  - 将处理后的内容写回EPUB文件

### 2. HTML 处理器 (HTMLProcessor)

HTML处理器采用两阶段处理策略，确保HTML内容的正确翻译：

#### 第一阶段：内容保护

- 使用占位符机制保护不需要翻译的内容：
  - 脚本和样式标签 (script, style)
  - 多媒体标签 (img, audio, video)
  - 表单元素 (input, button, textarea)
  - 特殊内容 (svg, math, canvas)
  - EPUB特有标签

#### 第二阶段：内容翻译

- 递归处理HTML节点：
  - 对于符合大小限制的节点，直接翻译
  - 对于超过限制的节点，递归处理其子节点
  - 翻译完成后还原占位符内容

### 3. 翻译服务

支持多种翻译提供者：

- Mistral
- OpenAI
- Google

每个提供者都实现了统一的翻译接口，支持：

- 文本翻译
- 速率限制
- 错误重试
- Token计数

## 工作流程

1. 用户上传EPUB文件
2. 系统创建工作目录并复制原文件
3. EPUB处理器加载文件
4. 提取HTML和NCX内容
5. HTML处理器处理每个内容文件：
   - 替换需保护的内容为占位符
   - 翻译文本内容
   - 还原占位符
6. 更新处理后的内容到EPUB文件
7. 保存并返回翻译后的文件
