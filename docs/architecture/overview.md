# EPUBox 系统架构概述

## 1. 系统目标

EPUBox是一个专注于EPUB电子书翻译的Web服务平台，支持多种翻译API接口，提供高质量的电子书翻译服务。

## 2. 核心功能

- EPUB文件解析和重构
- 多语言翻译支持
- 翻译进度追踪
- 断点续传
- 格式保持

## 3. 系统架构

### 3.1 整体架构

```
+----------------+     +------------------+     +------------------+
|                |     |                  |     |                  |
|  Web Interface |     | Translation Core |     |   EPUB Handler   |
|   (FastAPI)    |<--->|     Service     |<--->|                  |
|                |     |                  |     |                  |
+----------------+     +------------------+     +------------------+
                              ^
                              |
                      +-------v--------+
                      |                |
                      | Translation    |
                      |   Providers    |
                      |                |
                      +----------------+
```

### 3.2 主要组件

1. **Web Interface**
   - FastAPI Web服务
   - RESTful API接口
   - 任务状态查询

2. **Translation Core Service**
   - 翻译任务管理
   - 进度追踪
   - 断点续传支持

3. **EPUB Handler**
   - EPUB解析
   - 内容提取
   - 格式保持
   - EPUB重构

4. **Translation Providers**
   - 多API支持
   - 服务选择策略
   - 错误处理机制

## 4. 关键技术决策

### 4.1 不支持的功能
- 并发翻译
- 图片说明文字翻译
- 注释翻译
- 多用户任务管理

### 4.2 性能考虑
- 最大支持100M的EPUB文件
- 采用分章节处理策略
- 支持断点续传
- 实时进度显示

### 4.3 数据存储
- 任务状态持久化
- 翻译进度记录
- 检查点信息存储

## 5. 安全考虑

- API密钥安全存储
- 文件访问控制
- 错误处理和日志记录

## 6. 扩展性

系统设计考虑了以下扩展点：
- 新翻译服务提供商的接入
- 专业术语词典支持
- 翻译质量评估机制

## 7. 限制和约束

- 单任务处理
- 文件大小限制
- API调用限制
- 支持的文件格式
