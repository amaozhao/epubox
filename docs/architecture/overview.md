# EPUBox 系统架构概述

## 1. 系统目标

EPUBox是一个专注于EPUB电子书翻译的Web服务平台，支持多种翻译API接口，提供高质量的电子书翻译服务。

## 2. 核心功能

- EPUB文件解析和重构
- 基于树结构的HTML内容处理
- 专业词汇映射和一致性管理
- 多语言翻译支持
- 翻译进度追踪
- 断点续传
- 格式保持

## 3. 系统架构

### 3.1 整体架构

```
+----------------+     +------------------+     +------------------+
|                |     |                  |     |                  |
|  Web Interface |     | Translation Core |     |   EPUB Handler   |
|   (FastAPI)    |<--->|     Service     |<--->|                  |
|                |     |                  |     |                  |
+----------------+     +------------------+     +------------------+
                              ^
                              |
                      +-------v--------+     +------------------+
                      |                |     |                  |
                      | Translation    |<--->|  Tree Processor  |
                      |   Provider     |     |                  |
                      |                |     |                  |
                      +----------------+     +------------------+
```

### 3.2 主要组件

1. **Web Interface**
   - FastAPI Web服务
   - RESTful API接口
   - 任务状态查询
   - 进度实时更新

2. **Translation Core Service**
   - 翻译任务管理
   - 进度追踪
   - 断点续传支持
   - 错误恢复机制

3. **EPUB Handler**
   - EPUB解析和验证
   - 内容提取和分类
   - 格式保持和还原
   - EPUB重构和优化

4. **Translation Provider**
   - 翻译服务集成
   - 专业词汇管理
   - 翻译一致性保证
   - 错误处理和重试

5. **Tree Processor**
   - HTML树结构构建
   - 节点分类和合并
   - 文本块优化
   - 结构完整性维护

## 4. 关键技术决策

### 4.1 技术栈选择
- FastAPI作为Web框架
- SQLAlchemy 2.0用于数据持久化
- Pydantic v2用于数据验证
- BeautifulSoup用于HTML处理
- Structlog用于日志管理

### 4.2 不支持的功能
- 并发翻译
- 图片说明文字翻译
- 注释翻译
- 多用户任务管理

### 4.3 性能考虑
- 最大支持100M的EPUB文件
- 基于树结构的内容处理
- 智能文本块合并
- 专业词汇缓存
- 支持断点续传
- 实时进度显示

### 4.4 数据存储
- 任务状态持久化
- 翻译进度记录
- 检查点信息存储
- 专业词汇映射
